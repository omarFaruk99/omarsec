# File System

Everything in Linux is a file. Understanding the file system hierarchy is crucial.

## Directory Structure

```
/               # Root — the top of everything
├── bin/        # Essential command binaries (ls, cp, mv)
├── boot/       # Boot loader files
├── dev/        # Device files
├── etc/        # System configuration files
├── home/       # User home directories
│   └── omar/   # Your personal files
├── opt/        # Optional/third-party software
├── proc/       # Process information (virtual)
├── root/       # Root user's home directory
├── tmp/        # Temporary files (cleared on reboot)
├── usr/        # User programs and data
├── var/        # Variable data (logs, databases)
└── sbin/       # System binaries (admin commands)
```

## Key Directories You Should Know

| Directory | Purpose | Example |
|---|---|---|
| `/etc` | Config files | `/etc/nginx/nginx.conf` |
| `/var/log` | System logs | `/var/log/auth.log` |
| `/home` | User files | `/home/omar/Documents` |
| `/tmp` | Temp storage | Cleared on reboot |
| `/usr/bin` | User programs | Where installed apps live |

## File Types

```bash
ls -la
# drwxr-xr-x  → d = directory
# -rw-r--r--  → - = regular file
# lrwxrwxrwx  → l = symbolic link
```

## Important Paths

```bash
~              # Home directory shortcut
.              # Current directory
..             # Parent directory
/              # Root directory
```
