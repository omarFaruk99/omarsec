# Process Management

A process is a running instance of a program. Knowing how to manage them is critical for server administration and security work.

## Viewing Processes

```bash
ps                    # Show your processes
ps aux                # Show ALL processes from all users
top                   # Live process monitor (q to quit)
htop                  # Better version of top (install: sudo apt install htop)
```

## Process Information

```bash
ps aux | grep nginx   # Find a specific process
pidof nginx           # Get PID of a process by name
```

## Controlling Processes

### Foreground & Background

```bash
command &             # Run in background
jobs                  # List background jobs
fg %1                 # Bring job 1 to foreground
bg %1                 # Send job 1 to background
Ctrl + Z              # Suspend current process
Ctrl + C              # Kill current process
```

### Stopping Processes

```bash
kill PID              # Send SIGTERM (graceful stop)
kill -9 PID           # Send SIGKILL (force kill)
killall nginx         # Kill all processes by name
pkill -f "node server" # Kill by pattern match
```

## Systemd Services

```bash
sudo systemctl start nginx      # Start service
sudo systemctl stop nginx       # Stop service
sudo systemctl restart nginx    # Restart service
sudo systemctl status nginx     # Check service status
sudo systemctl enable nginx     # Start on boot
sudo systemctl disable nginx    # Don't start on boot
```

## Useful Monitoring

```bash
uptime                # How long system has been running
free -h               # Memory usage
df -h                 # Disk usage
lsof -i :80           # What's using port 80?
netstat -tlnp         # All listening ports
ss -tlnp              # Modern alternative to netstat
```
