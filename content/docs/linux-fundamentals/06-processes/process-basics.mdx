# Process Basics

<Callout type="info">
  তুমি যখন কোনো program চালাও, Linux সেটার জন্য RAM-এ জায়গা বরাদ্দ করে, CPU time দেয়, এবং একটি unique ID দেয় — এটাই process। System-এ কী চলছে সেটা না বুঝলে troubleshooting এবং cybersecurity দুটোই কঠিন।
</Callout>

## Process কী

**Program আর process এক জিনিস নয়।**

`python3` হলো disk-এ থাকা একটি file — এটা program।
তুমি যখন `python3 app.py` চালাও, Linux সেই file-কে RAM-এ load করে, CPU দিয়ে execute শুরু করে — এটা হলো process।

একই program থেকে একাধিক process হতে পারে:

```bash filename="Terminal"
python3 app.py &    # process 1 — PID 1001
python3 app.py &    # process 2 — PID 1002
python3 app.py &    # process 3 — PID 1003
# তিনটো আলাদা process, একই program থেকে
```

**প্রতিটি process-এর নিজস্ব পরিচয় আছে:**

| Property | মানে | কেন দরকার |
|----------|------|-----------|
| **PID** | Process ID — unique number | এই number দিয়েই process-কে target করা যায় |
| **PPID** | Parent Process ID | কে এই process তৈরি করেছে |
| **User** | কোন user-এর নামে চলছে | permission নির্ধারণ করে |
| **State** | এখন কী করছে | Running, Sleeping, Zombie, Stopped |
| **CPU/MEM** | কতটুকু resource নিচ্ছে | performance bottleneck খুঁজতে |

**PPID কেন গুরুত্বপূর্ণ:**

Linux-এ প্রতিটি process কোনো না কোনো process থেকে জন্ম নেয়। তুমি যখন terminal-এ `ls` চালাও:

```
terminal (bash) — PID 500
    └── ls — PID 501, PPID 500
```

`bash` হলো parent, `ls` হলো child। এই parent-child সম্পর্ক system-এর behavior বোঝার জন্য জরুরি।

## `ps`

`ps` command চলমান processes-এর snapshot দেখায়।

```bash filename="Terminal"
# শুধু current shell-এর processes
ps

# সব users-এর সব processes — সবচেয়ে বেশি ব্যবহার হয়
ps aux

# নির্দিষ্ট process খোঁজো
ps aux | grep nginx

# Parent-child relationship দেখো (tree format)
ps auxf
```

**`ps aux` এর output বোঝো:**

```
USER       PID  %CPU  %MEM    VSZ    RSS  TTY   STAT  START  TIME  COMMAND
root         1   0.0   0.1  16952   4096  ?     Ss    09:00  0:01  /sbin/init
omar      1234   0.5   1.2  123456  50000 pts/0  Sl   09:05  0:30  python3 app.py
```

| Column | মানে |
|--------|------|
| `USER` | কোন user-এর নামে চলছে |
| `PID` | Process ID |
| `%CPU` | এই মুহূর্তে CPU-র কত % ব্যবহার করছে |
| `%MEM` | Physical RAM-এর কত % ব্যবহার করছে |
| `VSZ` | Virtual memory — program যতটুকু চেয়েছে |
| `RSS` | Resident Set Size — আসলে RAM-এ কতটুকু আছে |
| `STAT` | Process-এর অবস্থা |
| `COMMAND` | কোন program চলছে |

<Callout type="default">
  **VSZ বনাম RSS:** VSZ হলো process-এর চাওয়া memory, RSS হলো আসলে ব্যবহার করা memory। অনেক সময় VSZ অনেক বড় দেখায় — ঘাবড়াবে না, RSS দেখো।
</Callout>

## `top`

`top` হলো live process monitor — প্রতি কয়েক সেকেন্ডে নিজে থেকে update হয়।

```bash filename="Terminal"
top
```

`top`-এর ভেতরে keyboard দিয়ে control করো:

| Key | কাজ |
|-----|-----|
| `q` | বের হও |
| `k` | Process kill করো (PID দিতে হবে) |
| `M` | Memory দিয়ে sort (কে বেশি RAM নিচ্ছে) |
| `P` | CPU দিয়ে sort (কে বেশি CPU নিচ্ছে) |
| `u` | নির্দিষ্ট user-এর processes দেখো |
| `1` | প্রতিটি CPU core আলাদা দেখাও |

## `htop`

`top`-এর উন্নত version। Color-coded, mouse support, scroll করা যায়।

```bash filename="Terminal"
# Install (না থাকলে)
sudo apt install htop

htop
```

<Callout type="default">
  **Practical note:** `top` সব Linux system-এ থাকে। `htop` install করতে হয় এবং remote server বা TryHackMe-এ নাও থাকতে পারে। `top` শেখাটাই বেশি কাজের।
</Callout>

## Process States

প্রতিটি process সবসময় একটি নির্দিষ্ট অবস্থায় থাকে:

| State | মানে | কেন হয় |
|-------|------|---------|
| `R` (Running) | CPU-তে execute হচ্ছে | normal কাজ |
| `S` (Sleeping) | কিছুর জন্য wait করছে | user input, timer |
| `D` (Uninterruptible) | Disk বা network-এর জন্য wait | I/O operation চলছে |
| `Z` (Zombie) | শেষ হয়েছে কিন্তু মুছে যায়নি | parent process এখনো exit code পড়েনি |
| `T` (Stopped) | ইচ্ছাকৃতভাবে থামানো | Ctrl+Z চাপা হয়েছে |

**Zombie process কেন হয়:**

যখন কোনো process শেষ হয়, সে সাথে সাথে মুছে যায় না। Linux সেটার exit code সংরক্ষণ করে রাখে যাতে parent process জানতে পারে — "কাজ হয়েছিল কি ব্যর্থ হয়েছিল?"

Parent process যদি সেই exit code না পড়ে (অথবা parent নিজেই crash করে), তাহলে child process zombie হয়ে আটকে থাকে।

```bash filename="Terminal"
# Zombie process দেখো
ps aux | grep Z
```

অল্প zombie স্বাভাবিক। অনেক zombie থাকলে কোনো program-এ bug আছে।

## Process খোঁজা

```bash filename="Terminal"
# নাম দিয়ে PID খোঁজো
pgrep nginx
pgrep python3

# নাম + full command দেখো
pgrep -a python3

# Parent-child tree দেখো
pstree -p

# কোন process কোন port ব্যবহার করছে
ss -tlnp | grep :80
```

## CyberSec Note

<Callout type="warning">
  **Process analysis — malware এবং rootkit detection:**

  **Unusual processes খোঁজা:**

  ```bash filename="Terminal"
  # কোন কোন user এর process চলছে দেখো
  ps aux | awk '{print $1}' | sort | uniq

  # Unexpected network connection কোন process করছে
  ss -tlnp

  # সম্প্রতি শুরু হওয়া processes
  ps aux --sort=start_time | tail -20
  ```

  **Rootkit detection — সবচেয়ে গুরুত্বপূর্ণ:**

  Linux-এ সব running process-এর জন্য `/proc/` directory-তে একটি folder থাকে। Rootkit সাধারণত `ps` command-কে manipulate করে নিজেকে লুকায় — কিন্তু `/proc/` পরিবর্তন করা কঠিন।

  ```bash filename="Terminal"
  # ps থেকে সব PID নাও
  ps aux | awk '{print $2}' | sort -n > /tmp/ps_pids

  # /proc থেকে সব PID নাও
  ls /proc | grep -E "^[0-9]+$" | sort -n > /tmp/proc_pids

  # তুলনা করো
  diff /tmp/ps_pids /tmp/proc_pids
  # /proc-এ আছে কিন্তু ps-এ নেই — rootkit suspect
  ```

  `ps` দেখাচ্ছে না কিন্তু `/proc`-এ দেখা যাচ্ছে — এই পার্থক্যটাই বলে দেয় যে `ps` কে manipulate করা হয়েছে।
</Callout>
