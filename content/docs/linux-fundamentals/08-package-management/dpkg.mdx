# dpkg

<Callout type="info">
  `apt` internet থেকে package download করে install করে। কিন্তু যদি manually একটা `.deb` file থাকে — যেমন GitHub থেকে download করা tool — তখন `dpkg` লাগে। CTF এবং pentesting-এ এটা প্রায়ই লাগে।
</Callout>

## apt vs dpkg

**apt এবং dpkg আসলে একে অপরের সাথে connected।**

`apt` আসলে ভেতরে `dpkg` ব্যবহার করে। তুমি যখন `apt install nmap` দাও:

```
apt → internet থেকে nmap.deb download করে
    → dpkg দিয়ে সেই .deb file install করে
```

তাই `dpkg` হলো lower-level tool। `apt` হলো সেটার উপরে সুবিধাজনক wrapper।

| | apt | dpkg |
|-|-----|------|
| Package source | Internet (repository) | Local `.deb` file |
| Dependency handle | হ্যাঁ — automatic | না — manually করতে হয় |
| Common use | নতুন software install | Downloaded .deb install |

## `.deb` File কোথা থেকে আসে

```bash filename="Terminal"
# GitHub release page থেকে download করা tool
wget https://github.com/tool/releases/latest/tool.deb

# কোনো vendor-এর official installer
wget https://example.com/software_1.0_amd64.deb

# apt দিয়ে শুধু download করো, install নয়
apt download nmap
# Current directory-তে nmap.deb চলে আসবে
```

## `dpkg` — Basic Commands

```bash filename="Terminal"
# .deb file install করো
sudo dpkg -i package.deb
# -i = install

# Package remove করো
sudo dpkg -r package-name
# -r = remove (config file রাখে)

# Package purge করো (config সহ)
sudo dpkg -P package-name
# -P = purge

# Installed packages-এর list
dpkg -l

# নির্দিষ্ট package installed কিনা
dpkg -l nmap
dpkg -s nmap    # বিস্তারিত status

# Package-এর কোন files কোথায় আছে
dpkg -L nmap
```

## Dependency Error ঠিক করা

**dpkg-র সবচেয়ে বড় সমস্যা:** dependency automatic handle করে না।

```bash filename="Terminal"
# dpkg install করার সময় error দিতে পারে:
sudo dpkg -i tool.deb
# dpkg: dependency problems prevent configuration of tool:
#  tool depends on libssl1.1; however:
#  Package libssl1.1 is not installed.

# সমাধান — apt দিয়ে missing dependency fix করো
sudo apt install -f
# -f = fix broken
```

`apt install -f` চালালে apt বুঝতে পারে কোন dependency missing এবং সেগুলো install করে দেয়।

## Real Example — Burp Suite Install

Burp Suite (web pentesting tool) official site থেকে `.deb` file হিসেবে আসে:

```bash filename="Terminal"
# Step 1: Download করো
wget https://portswigger.net/burp/releases/download -O burpsuite.deb

# Step 2: dpkg দিয়ে install করো
sudo dpkg -i burpsuite.deb

# Step 3: Dependency error হলে fix করো
sudo apt install -f

# Step 4: চালাও
burpsuite
```

## Package Information দেখা

```bash filename="Terminal"
# Package installed কিনা এবং version
dpkg -s nmap
# Package: nmap
# Status: install ok installed
# Version: 7.93+dfsg1-1

# Package-এর সব files কোথায় আছে
dpkg -L nmap
# /usr/bin/nmap
# /usr/share/man/man1/nmap.1.gz
# ...

# কোন package এই file-টা install করেছে
dpkg -S /usr/bin/nmap
# nmap: /usr/bin/nmap
```

## CyberSec Note

<Callout type="warning">
  **dpkg — CTF এবং pentesting-এ:**

  **Target machine-এ installed software version খোঁজো:**

  ```bash filename="Terminal"
  # সব installed packages
  dpkg -l

  # নির্দিষ্ট software-এর version
  dpkg -l | grep -i "apache2\|nginx\|openssh\|php"

  # Exact version দেখো
  dpkg -s apache2 | grep Version
  # Version: 2.4.41-4ubuntu3
  ```

  এই version দিয়ে CVE database-এ (`cve.mitre.org`, `exploit-db.com`) known vulnerability খোঁজো।

  **Malicious .deb:** Attacker কখনো কখনো malicious `.deb` file তৈরি করে। `dpkg` install করার সময় post-install script চলে — এখানে malicious code থাকতে পারে। অবিশ্বস্ত `.deb` file কখনো install করবে না।
</Callout>
